Como o computador transforma linguagem de alto nível em assembly e depois em linguagem de máquina?

A Jornada do Código

Os computadores são incapazes de compreender diretamente as linguagens de programação que utilizamos no dia a dia. Para que um programa escrito em uma sintaxe humana funcione, ele precisa passar por um processo rigoroso de tradução. Essa transformação ocorre em etapas, reduzindo gradualmente a abstração do código até que ele se torne um conjunto de instruções puramente numéricas que o processador consiga executar.

Do alto nível ao Assembly

Para converter uma linguagem de alto nível para Assembly, o compilador precisa passar por 4 etapas base, descritas abaixo, para poder fornecer um código funcional da linguagem de baixo nível.

Tendo como base a linguagem C, podemos utilizar de um coamndo do terminal para realizar essa conversão e mostrar ao usuário o código gerado:

gcc -S programa.c

Seguindo a instrução acima, o compilador segue as seguintes operações para gerar o código Assembly:

1. Pré-processamento

Nessa etapa, o compilador remove os comentários dos códigos e interpreta os macros, mudando as constantes definidas com #define e incluindo os arquivos dos #include.

2. Análise sintática

Verificação do código, nessa parte ocorre o encontro de erros e os comandos são mapeados e ordenados para que a lógica seja seguida como o esperado.

3. Otimização

Nessa etapa, o compilador procura formas melhores de executar certas funções sem quebrar a lógica do progrma, garantindo maior rapidez e eficiência.

4. Conversão

Por fim, o compilador traduz o código otimizado em linguagem de baixo nível, Assembly, transformando cada instrução em diferentes linhas da linguagem.

Do Assembly à linguagem de máquina

Agora que temos o Assembly do código, precisamos entender como ele é transformado em linguagem de máquina. Para isso, é utilizado do montador, ou assembler.

Toda instrução em Assembly possui uma codificação em binário(0s e 1s), que representa a operação e, caso necessário, os valores imaediatos e registradores usados. Dessa forma, o montador segue uma série de passos para definir o comando a ser feito e quais valores utilizar.

1. Leitura do Opcode

O montador identifica a operação desejada a partir do Opcode(código de operação) e transforma o comando em seu binário equivalente.

2. Tradução dos operandos

Além da conversão do Opcode para binário, o monttador é responsável pela transformação de registradores, endereços de memória e valores numéricos em formatos binários que representem o dado.

3. Cálculo dos endereços

Calcula a distância dos endereços de memória necessários durante a operação por meio da determinação de posições de memórias para labels e jumps.
